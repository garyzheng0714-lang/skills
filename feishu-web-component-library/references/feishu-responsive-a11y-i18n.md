# 响应式、无障碍与国际化规则

## 1. 响应式规则

### 1.1 断点
- XS: `0-599`
- S: `600-1023`
- M: `1024-1439`
- L: `1440-1919`
- XL: `>=1920`

### 1.2 页面类型
- 全屏型：内容区随屏幕增长而扩展，适合表格/图表等高信息密度页面。
- 定宽型：内容区到达上限后保持固定（推荐 1136），适合聚焦型任务（如表单、官网信息页）。

### 1.3 组件映射
- 顶部导航：完整 -> 半收起 -> 图标入口。
- 侧边导航：完整 -> 折叠栏 + popover -> 抽屉。
- 按钮组/Tabs：完整展示 -> 收起到“更多”。
- Dialog：固定档位 -> 响应式宽度 + 内部滚动。
- Table：固定列/表头 + 横向滚动，必要时简化列。

### 1.4 密度策略
- 高密度：批量浏览、批量编辑、对比分析。
- 低密度：警示、确认、关键操作、触控密集场景。

## 2. 无障碍规则（A11y）

### 2.1 视觉可读性
- 按钮文本与背景对比度达到 `4.5:1`（禁用态可例外）。
- 卡片与背景应有可识别分层（对比、描边或投影）。

### 2.2 键盘可达性
- Button: Tab/Enter/Space。
- Dialog: Tab 焦点陷阱、Esc 关闭。
- 菜单/列表：方向键切换，Enter 触发。
- 图片预览类场景：方向键导航，Esc 退出（文档中有明确示例）。

### 2.3 ARIA 最低要求
- Button: `aria-pressed`, `aria-disabled`, `aria-busy`（按场景）。
- Dialog: `role=dialog`, `aria-modal`, `aria-labelledby`。
- Tooltip: `role=tooltip`, `aria-describedby`。
- Table/Grid: `role=table|grid`, `aria-sort`, `aria-rowcount`。

### 2.4 图标语义
- 图标按钮建议强制提供 Tooltip 或 `aria-label`。
- 图标仅在含义明确时使用，避免线面风格混用。

## 3. 国际化规则（i18n）

### 3.1 文案扩展
- 设计阶段提前预留多语言横向空间，不以单语（尤其中文）长度做唯一基准。
- 超长文案优先方案：改布局 > 折行 > 缩字号（最后手段）。

### 3.2 字体高度差异
- 需考虑三类文字系统：类英语、密集文字、高文字。
- 泰语/越南语/印地语等高文字需要更宽松行高，避免上下裁切。

### 3.3 折行规则
- 西文按单词边界折行。
- 日文/泰文等按语义边界处理，必要时人工指定换行点。

### 3.4 布局建议
- 长文案场景优先单列纵向布局。
- 移动端每行建议 3-4 列，超过部分可横滑或折叠。

## 4. 本 Skill 的执行策略
- 默认先应用响应式与 i18n 约束，再生成组件代码。
- 如出现冲突，优先级：可用性 > 信息完整性 > 视觉一致性。
- 需要极端文案验证时运行：`scripts/simulate_i18n_expansion.py`。
