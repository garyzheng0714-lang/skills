# 飞书登录 UI 统一规范（FBIF 默认皮肤）

## 强制不可改原则

任何项目接入飞书登录时，登录入口的视觉/文案/交互必须严格遵守本规范。AI 不得"自由发挥"换颜色、改字号、换 logo 大小、改文案。**原因：跨项目体验一致性**——用户每个 FBIF 系统看到的都是一样的登录入口，不会"这个项目飞书蓝那个项目深蓝、这个写'飞书登录'那个写'登录'"。

如果项目已有设计系统强制冲突（必须用 Tailwind tokens 或 CSS Modules 而不能 inline style），按本规范的**视觉值**保留，只把实现方式换。视觉值不变。

---

## 主品牌：FBIF（默认）

本 skill 默认是 FBIF 项目的登录入口。如果接入的是非 FBIF 项目，把 `assets/fbif-logo.webp` 替换成项目品牌 logo、修改组件的 `appName` / 文案 props 即可。**视觉规格不变**。

### FBIF Logo

源文件：
- `assets/fbif-logo.webp`（5KB，主用）
- `assets/fbif-logo.png`（6KB，备用兼容）

实现要求：
- **登录页顶部使用 webp**，通过 `<img src="/fbif-logo.webp" />` 加载（部署时把文件 cp 到项目 `public/` 目录）
- 在登录页展示尺寸：**88px × 88px**（视觉重心）
- 顶栏 / 小标识展示尺寸：32px × 32px
- 不要倒转、不要改色、不要描边

### FBIF 品牌色

| 用途 | 值 |
|---|---|
| 主色（按钮背景、CTA） | `#145078` |
| 主色 hover/active | `#0F3D5C` |
| 强调色（仅 logo 内的感叹号） | `#F0C800` |
| 标题文字 | `#0F1F33`（深蓝近黑） |
| 正文 | `#5A6B80`（中灰蓝） |
| 辅助文字 | `#7A8AA0`（浅灰蓝） |
| 边框 | `#E5E9F0` |
| 卡片背景 | `#FFFFFF` |
| 页面背景 | `#FAFBFC`（极浅灰，不刺眼） |

**核心原则**：浅色背景 + 深蓝品牌色 + 中性灰阶。**绝不用纯黑底（#000）或暗色主题**——这是用户明确要求。

## 副品牌：飞书

仅在登录按钮内出现飞书 logo（小尺寸，让用户知道是走飞书 OAuth）。

### 飞书 Logo

源文件：`assets/feishu-logo.svg`

实现要求：
- **必须 inline 进 React 组件**（不用 import 路径），避免打包/部署时加载失败
- 完整保留三色（绿/蓝/深蓝），不要单色化
- 仅出现在登录按钮内、与文字基线对齐、间距 10px

## 路由 / URL 约定（跨项目一致，永不变）

| 路径 | 用途 |
|---|---|
| `/login` | 登录页（前端，渲染 FeishuLoginPage） |
| `GET /auth/feishu/login` | 触发登录（302 跳飞书） |
| `GET /auth/feishu/callback` | 飞书授权回调（处理 code） |
| `POST /auth/feishu/logout` | 退出登录 |
| `GET /api/me` | 当前用户 JSON，未登录返 401 |

`redirect_uri` 在飞书后台配的是 `<部署URL>/auth/feishu/callback`。**永远不换路径**。

## 错误状态

| 错误 | 中文 | URL query | 处理 |
|---|---|---|---|
| 用户拒绝授权 | 已取消授权，可重新登录 | `?error=denied` | 显示后用户可点登录重试 |
| state 不匹配 | 安全校验失败，请刷新重试 | （不展示给用户） | 后端直接 403 |
| tenant 不在白名单 | 你的飞书账号不在授权范围内，请用 FBIF 账号登录 | `?error=unauthorized_tenant` | 飞书自带的拦截页 + 我们前端兜底 |
| 其他 | 登录失败，请稍后重试 | `?error=other` | 重试链接 |

## 不要做的事

- ❌ **不要用暗黑色背景**（#000、#1a1a1a 这类）。用户明确说不要。
- ❌ 不要把按钮颜色换成飞书蓝 `#3370FF`（按钮要 FBIF 蓝 `#145078`，飞书蓝只在按钮内的 logo 自带）
- ❌ 不要改文案"使用 FBIF 登录"为"使用飞书登录"或"立即登录"
- ❌ 不要换 logo 尺寸（88px 是登录页固定值）
- ❌ 不要把卡片宽度做太宽（480px 是上限，再宽就不像登录卡片了）
- ❌ 不要加多个登录方式（QQ/微信/Google 等）—— 本 skill 仅支持飞书 OAuth

## 完整组件实现

`templates/frontend/react-vite/FeishuLogin.tsx.template` 含：
- `FeishuLoginPage` —— drop-in 整页登录（默认）
- `FeishuLoginButton` —— 单按钮（自定义页用）
- `useCurrentUser` —— hook
- `RequireAuth` —— 受保护路由包裹
- `FeishuUserBadge` —— 顶栏当前用户

适配其他前端框架（Vue / Svelte / Solid）时，按本规范的视觉值翻译即可。
